A modular Python framework for estimating the **intrinsic predictability**
of time series using statistical analysis and optional machine learning
backends.

Instead of asking

> **"Can my model forecast this series?"**

Predictability Score asks

> **"Is this series fundamentally predictable?"**

The framework separates statistical properties of a time series from the
capabilities of any particular forecasting model.

---

## Why Predictability Score?

Many forecasting projects fail before model training even begins.

A forecasting model may perform poorly because

- the series is non-stationary,
- temporal dependencies are weak,
- observed patterns are generated by randomness,
- statistical properties change over time,
- or the forecasting model cannot exploit the available information.

Predictability Score evaluates these properties independently and combines
them into a single interpretable score.

# Predictability Scale

| Score | Interpretation |
|--------|----------------|
| 0 – 20 | Very Low Predictability |
| 20 – 40 | Low Predictability |
| 40 – 60 | Moderate Predictability |
| 60 – 80 | High Predictability |
| 80 – 100 | Very High Predictability |

The score should be interpreted as an estimate of the **intrinsic
predictability** of the observed time series rather than the expected
performance of any specific forecasting model.

---

# Evaluation Architecture

The overall score is computed as a weighted combination of five independent
components.

| Component | Weight |
|------------|-------:|
| Stationarity | 20% |
| Temporal Dependence | 25% |
| Forecastability | 30% |
| Null Significance | 15% |
| Stability | 10% |

Mathematically,

```text
Predictability Score

= 0.20 × Stationarity
+ 0.25 × Temporal Dependence
+ 0.30 × Forecastability
+ 0.15 × Null Significance
+ 0.10 × Stability
```

Each component measures a different aspect of predictability and is designed
to complement the others.

No single statistical test determines the final score.

---

# Core Components

## 1. Stationarity

Measures whether the statistical properties of the series remain stable over
time.

Implemented methods

- Augmented Dickey–Fuller (ADF)
- KPSS

Output

```text
0 ←──────────────→ 100
Non-stationary      Stationary
```

---

## 2. Temporal Dependence

Measures whether meaningful temporal structure exists independently of any
forecasting model.

Implemented methods

- Ljung–Box Test
- BDS Test
- Durbin–Watson Statistic

This component answers

> Does the series contain exploitable temporal information?

---

## 3. Forecastability

Measures how well a forecasting backend can predict unseen observations.

Current backend

- TensorFlow LSTM

Future backends

- PyTorch
- TCN
- Transformer
- CatBoost
- XGBoost
- LightGBM

Current evaluation metric

- Out-of-sample R²

---

## 4. Null Significance

Measures whether detected temporal structure is statistically significant.

Method

Permutation testing

Procedure

1. Shuffle the time series.
2. Destroy temporal ordering.
3. Recompute temporal dependence.
4. Build a null distribution.
5. Compare the observed score with the null distribution.

A high score indicates that the observed temporal structure is unlikely to
have arisen by chance.

---

## 5. Stability

Measures whether predictability remains consistent over time.

The series is divided into multiple windows.

Each window is evaluated independently.

Stable series exhibit

- consistent scores,
- low variability,
- persistent statistical behavior.

# Quick Start

Evaluating a time series requires only a few lines of code.

```python
import numpy as np

from predictability_score import PredictabilityScore

series = np.random.randn(5000)

evaluator = PredictabilityScore()

result = evaluator.evaluate(series)

print(result)
```

# Understanding the Score

The overall score summarizes several complementary properties of a time series.

A high score does **not** guarantee that every forecasting model will perform well.

Likewise, a low score does **not** necessarily imply that forecasting is impossible.

Instead, the score estimates the intrinsic predictability of the observed series.

Typical interpretation

| Score | Interpretation |
|-------:|----------------|
| 0–20 | Almost unpredictable |
| 20–40 | Weak predictive structure |
| 40–60 | Moderate predictability |
| 60–80 | Strong predictive structure |
| 80–100 | Highly predictable |

# Financial Time Series Example

Financial price series are usually **non-stationary**.

Instead of raw prices, it is recommended to evaluate transformed variables such as log returns.

```python
import numpy as np

returns = np.log(close / close.shift(1))

returns = returns.dropna()

evaluator = PredictabilityScore()

result = evaluator.evaluate(returns)

print(result)
```

Other useful financial transformations include

- Log Returns
- Realized Volatility
- ATR-based Features
- High/Open
- Low/Open
- Normalized OHLC
- Spread
- Volume Features

# Using the TensorFlow Backend

The statistical framework works without TensorFlow.

To enable model-based forecastability evaluation:

```python
from predictability_score import (
    PredictabilityConfig,
    PredictabilityScore
)

config = PredictabilityConfig(
    forecast_backend="tensorflow"
)

evaluator = PredictabilityScore(config)

result = evaluator.evaluate(series)
```

When enabled, the TensorFlow backend

- builds a lightweight LSTM model,
- trains using historical observations,
- evaluates out-of-sample performance,
- converts predictive performance into a normalized Forecastability score.

# What Makes Predictability Score Different?

Many existing tools answer questions such as

- Is the series stationary?
- Is there autocorrelation?
- Does an LSTM obtain good accuracy?

Predictability Score combines these perspectives into a unified framework.

| Existing Approach | Predictability Score |
|-------------------|----------------------|
| Single statistical test | Multiple complementary analyses |
| Model-specific evaluation | Model-independent evaluation |
| One metric | Composite score |
| Fixed methodology | Modular backend architecture |
| Statistical tests only | Statistics + Machine Learning |

---

# Design Philosophy

Predictability Score is built around three core principles.

## 1. Separate Statistical Structure from Model Performance

A time series may contain strong temporal structure while remaining difficult for a particular forecasting model.

Therefore,

```text
Temporal Dependence

≠

Forecastability
```

These concepts are evaluated independently.

---

## 2. Machine Learning Should Be Optional

Statistical evaluation should not require TensorFlow, PyTorch, or GPU support.

The core package depends only on scientific Python libraries.

Forecasting backends are optional extensions.

---

## 3. Extensibility

The framework is designed around interchangeable forecasting backends.

New models can be added without modifying the statistical core.
---

# Architecture

Predictability Score follows a modular architecture designed around separation
of concerns.

The framework consists of three main layers:

```text
                    User Application

                           |
                           v

              +------------------------+
              |  PredictabilityScore   |
              |      Core Engine       |
              +------------------------+

                           |
        +------------------+------------------+
        |                  |                  |
        v                  v                  v

+---------------+  +---------------+  +---------------+
| Statistical   |  |  Significance |  |   Stability   |
| Evaluators    |  |    Testing    |  |   Analysis    |
+---------------+  +---------------+  +---------------+

                           |
                           v

              +------------------------+
              | Forecasting Backend   |
              |  (Optional Layer)     |
              +------------------------+

                           |
              +------------+-------------+
              |                          |
              v                          v

        TensorFlow LSTM              Future Models

                                  - PyTorch
                                  - TCN
                                  - Transformer
                                  - Classical ML
```

The architecture allows:

- statistical evaluation without machine learning dependencies,
- independent forecasting engines,
- future extension without modifying the core framework.

# FAQ

## Does Predictability Score predict the future?

No.

It evaluates whether a time series contains properties that make forecasting
more feasible.

A forecasting model is still required to generate future predictions.

---

## Does a high score guarantee profitable trading?

No.

Financial predictability and trading profitability are different concepts.

A predictable series may still have:

- transaction costs,
- market constraints,
- poor risk/reward characteristics.

---

## Is TensorFlow required?

No.

The core statistical framework works independently.

TensorFlow is only required for the optional forecasting backend.

---

## Can I add my own forecasting model?

Yes.

The backend architecture is designed for extensions.

Future implementations can include:

- PyTorch,
- Transformers,
- TCN,
- classical machine learning models.
